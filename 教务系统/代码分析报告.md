## 一、侧边栏共享机制：为什么 13 个页面只用维护 1 份 HTML

### 1.1 问题

多页面架构下，13 个 HTML 文件各有完全相同的顶栏 + 侧边栏 + 下拉菜单。每次新增一个菜单项（比如加了「考试安排」页面），需要手动修改全部 13 个文件——这导致开发过程中反复出现标签闭合错误（`<button>` 开头 `</a>` 结尾）。

### 1.2 解决方案

采用**模板 + 占位符替换**的模式。整个外壳只写一次（`_shell.html`），每个页面通过 `_shell.js` 动态加载并注入。

### 1.3 _shell.html：模板文件

```html
<!-- _shell.html — 只写一次的公共外壳 -->

<!-- 顶栏 -->
<header class="topbar">
  <div class="topbar-left">
    <button class="hamburger" id="hamburgerBtn">...</button>
    <a href="index.html" class="topbar-logo">教务系统</a>
    <nav class="breadcrumb">{{BREADCRUMB}}</nav>     <!-- ← 每页不同 -->
  </div>
  <div class="topbar-right">
    <button class="topbar-btn" id="darkModeToggle">...</button>
    <a href="notifications.html" class="topbar-btn">
      <span class="notification-count" id="topNotifCount">{{BADGE}}</span>  <!-- ← 动态未读数 -->
    </a>
    <div class="user-menu-wrap">
      <button class="user-menu" id="userMenuBtn">
        <div class="user-avatar" id="shellAvatar">李</div>
        <span class="user-name" id="shellUserName">李小宇</span>
      </button>
      <div class="user-dropdown" id="userDropdown">...</div>
    </div>
  </div>
</header>

<!-- 侧边栏 -->
<aside class="sidebar" id="sidebar">
  <nav class="sidebar-nav">
    <a href="index.html" class="nav-item{{ACTIVE:index}}">首页仪表盘</a>       <!-- ← {{ACTIVE:index}} -->
    <a href="schedule.html" class="nav-item{{ACTIVE:schedule}}">课表</a>      <!-- ← {{ACTIVE:schedule}} -->
    <a href="grades.html" class="nav-item{{ACTIVE:grades}}">成绩查询</a>      <!-- ← {{ACTIVE:grades}} -->
    ... (共 12 个菜单项)
  </nav>
</aside>
```

**三种占位符**：

| 占位符 | 含义 | 替换逻辑 |
|:---|:---|:---|
| `{{ACTIVE:index}}` | 当前页高亮标记 | 匹配则替换为 ` active`，不匹配则替换为空 |
| `{{BREADCRUMB}}` | 面包屑路径 | 替换为每页的路径文字（如 `首页 / 课表`） |
| `{{BADGE}}` | 未读 Badge 数字 | 替换为 `localStorage.getItem('unreadCount')` |

### 1.4 _shell.js：加载器

```javascript
(function() {
  // 第一步：每个页面在 <html> 标签上标注自己的身份
  // <html data-shell="schedule" data-breadcrumb="首页 / 课表">
  var activePage = document.documentElement.getAttribute('data-shell') || 'index';
  var breadcrumb = document.documentElement.getAttribute('data-breadcrumb') || '首页';

  // 第二步：用 fetch 拉取模板
  fetch('_shell.html')
    .then(function(r) { return r.text(); })
    .then(function(html) {

      // 第三步：正则替换 {{ACTIVE:page}} → " active" 或 ""
      html = html.replace(/\{\{ACTIVE:(\w[-\w]*)\}\}/g, function(m, page) {
        return page === activePage ? ' active' : '';
      });

      // 第四步：替换面包屑和 Badge
      html = html.replace(/\{\{BREADCRUMB\}\}/g, breadcrumb);
      html = html.replace(/\{\{BADGE\}\}/g, localStorage.getItem('unreadCount') || '3');

      // 第五步：注入到页面的 <div id="app-shell"> 占位容器
      document.getElementById('app-shell').innerHTML = html;

      // 第六步：初始化所有共享交互（暗黑模式、侧边栏折叠、下拉菜单等）
      initShared();
    });
})();
```

### 1.5 每个页面的接入方式

```html
<!-- 在 <html> 标签上声明身份 -->
<html lang="zh-CN" data-shell="schedule" data-breadcrumb="首页 / 课表">

<!-- 页面 body 中只需要一个容器 -->
<body>
  <div id="app-shell"></div>         <!-- ← 外壳注入到这里 -->
  <div class="main-content">
    <!-- 页面独有的内容 -->
  </div>
  <script src="_shell.js"></script>  <!-- ← 自动加载外壳 -->
</body>
```

### 1.6 设计意图

这套机制的核心思想是**声明式配置 + 自动注入**。每个页面只需要声明"我是谁"（`data-shell`）和"我在哪"（`data-breadcrumb`），外壳的 HTML 结构和交互逻辑由 `_shell.js` 统一管理。

对比传统做法：如果不用这套机制，13 个页面 × 约 80 行侧边栏 HTML = 1040 行需要手动维护的重复代码。任何一次菜单调整（比如加一个「考试安排」）都需要改 13 个文件。现在只需要改 `_shell.html` 一处。

## 二、跨页面状态管理：localStorage 持久化矩阵

### 2.1 为什么需要跨页面状态

多页面架构下，每个页面加载时都是"全新"的——之前用户在通知页标记了已读，切换到首页时首页的 Badge 仍然是硬编码的 `3`。localStorage 是连接这 13 个独立页面的"共享内存"。

### 2.2 完整状态矩阵

```
┌─────────────────────┬──────────┬────────────────────┬──────────────────────┐
│ localStorage 键     │ 数据类型 │ 写入方             │ 读取方               │
├─────────────────────┼──────────┼────────────────────┼──────────────────────┤
│ theme               │ String   │ 任一点击           │ 全部 13 页           │
│ unreadCount         │ Number   │ notifications.html │ 全部页面 Badge       │
│ readNotifIds        │ [1,4,5]  │ notifications.html │ 通知页+首页红点      │
│ pinnedNotifIds      │ [2,3]    │ notifications.html │ 通知页置顶排序       │
│ userName            │ String   │ personal.html      │ 全部页面顶栏+横幅    │
│ courseCart          │ [id,id]  │ course-selection   │ 选课页购物车恢复     │
│ submittedIds        │ [id,id]  │ course-selection   │ 选课页取消提交       │
│ courseSeats         │ {id:n}   │ course-selection   │ 选课页余量恢复       │
│ autoCollapse        │ Boolean  │ settings.html      │ 侧边栏默认折叠       │
└─────────────────────┴──────────┴────────────────────┴──────────────────────┘
```

### 2.3 通知状态持久化（以通知页为例）

**数据存储**：每条通知的已读状态用 `readNotifIds` 数组记录（只存已读的 ID，节省空间）。

```javascript
// 每次渲染时，收集所有已读 ID → 存入 localStorage
function renderNotifications() {
  // ... 过滤和排序逻辑 ...

  // 收集已读通知的 ID
  var readIds = NOTIFICATIONS
    .filter(function(n) { return n.read; })
    .map(function(n) { return n.id; });

  // 写入 localStorage
  localStorage.setItem('readNotifIds', JSON.stringify(readIds));
  localStorage.setItem('unreadCount', NOTIFICATIONS.filter(function(n) { return !n.read; }).length);
}
```

**数据恢复**：页面加载时，读取 `readNotifIds` 恢复到 NOTIFICATIONS 数组。

```javascript
// 页面初始化
var readIds = JSON.parse(localStorage.getItem('readNotifIds') || '[]');

NOTIFICATIONS.forEach(function(n) {
  // 如果该通知的 ID 在 readIds 数组中 → 标记为已读
  n.read = (readIds.indexOf(n.id) !== -1);
});
```

### 2.4 首页通知红点同步

首页的通知卡片有 3 条展示，每条有 `data-nid` 属性。通过读取同一个 `readNotifIds`，决定是否显示红点：

```javascript
// index.html 初始化时
var readIds = JSON.parse(localStorage.getItem('readNotifIds') || '[]');

document.querySelectorAll('#homeNotifList .notif-item').forEach(function(item) {
  var nid = parseInt(item.dataset.nid);
  // ID 在已读数组中 → 移除 unread 类（隐藏红点）
  // ID 不在已读数组中 → 保持 unread 类（显示红点）
  if (readIds.indexOf(nid) !== -1) {
    item.classList.remove('unread');
  } else {
    item.classList.add('unread');
  }
});
```

两个页面（notifications.html 和 index.html）共享同一个 `readNotifIds` 数据源。通知页标记已读 → 写 localStorage → 首页读取 → 红点消失。全程不需要后端 API。

### 2.5 选课购物车持久化（以选课页为例）

**三合一持久化**：选课页的状态最复杂，涉及三个相互关联的数据。

```javascript
function persistState() {
  // ① 购物车：存课程 ID 数组
  localStorage.setItem('courseCart', JSON.stringify(
    cart.map(function(c) { return c.id; })
  ));

  // ② 已提交 ID：用于取消提交
  localStorage.setItem('submittedIds', JSON.stringify(submittedIds));

  // ③ 课程余量：存 ID → seats 的映射
  var seatsMap = {};
  ALL_COURSES.forEach(function(c) { seatsMap[c.id] = c.seats; });
  localStorage.setItem('courseSeats', JSON.stringify(seatsMap));
}

function restoreState() {
  // ① 恢复购物车：ID → 课程对象
  var ids = JSON.parse(localStorage.getItem('courseCart') || '[]');
  cart = ids.map(function(id) {
    return ALL_COURSES.find(function(c) { return c.id === id; });
  }).filter(Boolean);

  // ② 恢复已提交 ID
  submittedIds = JSON.parse(localStorage.getItem('submittedIds') || '[]');

  // ③ 恢复余量
  var sm = JSON.parse(localStorage.getItem('courseSeats') || '{}');
  ALL_COURSES.forEach(function(c) {
    if (sm[c.id] !== undefined) c.seats = sm[c.id];
  });
}
```

调用 `persistState()` 的时机：加入购物车、移除购物车、提交、取消提交——每次状态变更都立即写入。

## 三、暗黑模式：CSS Variables 全局切换机制

### 3.1 实现原理

暗黑模式不是简单的"颜色取反"，而是**精细化变量映射**。

```css
:root {
  /* 亮色模式基准 */
  --color-neutral-50:  #F8FAFC;   /* 页面底色 */
  --color-neutral-100: #F1F5F9;   /* 卡片底色 */
  --color-neutral-800: #1E293B;   /* 标题文字 */
  --color-primary-500: #3B82F6;   /* 主色 */
}

body.dark {
  /* 暗色模式：中性色翻转明暗，蓝色向浅色偏移 */
  --color-neutral-50:  #0F172A;   /* 深色底 */
  --color-neutral-100: #1E293B;   /* 暗卡片 */
  --color-neutral-800: #F1F5F9;   /* 亮文字（翻转！） */
  --color-primary-500: #60A5FA;   /* 提亮一档 */
}
```

关键点：只改变量值，不改变量名。页面中所有地方引用的是 `var(--color-neutral-50)`，而不是具体色值。切换暗黑模式时只需要给 `<body>` 加一个 `dark` 类，所有引用自动生效。

### 3.2 切换逻辑

```javascript
function setDarkMode(on) {
  document.body.classList.toggle('dark', on);

  // 同步更新图标（月亮 ↔ 太阳）
  darkIcon.innerHTML = on
    ? '<circle.../>...'   // 太阳图标
    : '<path.../>';       // 月亮图标

  // 持久化偏好
  localStorage.setItem('theme', on ? 'dark' : 'light');
}

// 页面加载时读取偏好
if (localStorage.getItem('theme') === 'dark') setDarkMode(true);
```

### 4.1 首页学分环形图

这是首页最核心的数据展示。使用 ECharts 的 Pie 类型 + `graphic` 组件。

```javascript
var creditChart = echarts.init(document.getElementById('creditRing'));

creditChart.setOption({

  graphic: [{
    type: 'text',

      fill: '#1E293B'
    }
  }],

  series: [{
    type: 'pie',
    radius: ['70%', '88%'],    // 环形：内径 70%，外径 88%
    silent: true,               // 不响应鼠标
    label: { show: false },
    data: [
      { value: 118,              // 已修学分
        itemStyle: {
          color: new echarts.graphic.LinearGradient(0, 0, 1, 1, [
            { offset: 0, color: '#3B82F6' },    // 蓝
            { offset: 1, color: '#7C3AED' }     // → 紫渐变

      },

```

**为什么用 `graphic` 而不是 Gauge？** 最初用了 Gauge 的 `startAngle=0, endAngle=0` hack 来显示中心文字，但在部分 ECharts 版本中渲染异常。`graphic` 是 ECharts 原生的文字绘制组件，兼容性好且不依赖 Series 渲染。

## 五、选课系统事务逻辑

### 5.1 完整状态机

```
        addToCart()
  [空购物车] ──────────→ [购物车有课]
     ↑                      │
     │              submitCart() → [等待结果]
     │                  │              │
     │          ┌───────┴───────┐      │
     │          │ 80% 成功      │ 20% 失败
     │          │ seats-1       │ 保留在购物车
     │          │ submittedIds  │
     │          └───────┬───────┘      │
     │                  ↓              │
     │         undoSubmit()            │
     │          seats+1                │
     │          退回购物车 ←───────────┘
     │                  │
     └──── removeFromCart()
          退选单门课程
```

### 5.2 提交与取消提交核心代码

```javascript
function submitCart() {
  // 模拟 80% 成功率
  var ok = [], fail = [];
  cart.forEach(function(c) {
    if (Math.random() > 0.2) { ok.push(c); } else { fail.push(c); }
  });

  // 成功的课程：余量 -1，记录到 submittedIds
  ok.forEach(function(c) {
    var co = ALL_COURSES.find(function(x) { return x.id === c.id; });
    if (co && co.seats > 0) { co.seats -= 1; submittedIds.push(c.id); }
  });

  persistState();  // 三合一写入 localStorage
  cart = fail;     // 失败的留在购物车
  updateAll();
}

function undoSubmit() {
  // 遍历已提交 ID，恢复余量，退回购物车
  submittedIds.forEach(function(id) {
    var co = ALL_COURSES.find(function(c) { return c.id === id; });
    if (co) { co.seats += 1; cart.push(co); }
  });
  submittedIds = [];
  persistState();
  updateAll();
}
```

## 六、课表动态日期计算

```javascript
// 学期基准：2026年6月8日（周一）= 第 16 周
var WEEK_START_DATE = new Date(2026, 5, 8);

// 当前时间与基准日的差值
var nowDate = new Date();
var dayOffset = Math.floor((nowDate - WEEK_START_DATE) / 86400000);

// 动态计算当前是第几周、星期几
var CURRENT_WEEK = 16 + Math.floor(dayOffset / 7);
var CURRENT_DAY = nowDate.getDay() === 0 ? 7 : nowDate.getDay();  // Mon=1, Sun=7
```

然后在渲染周视图网格时：

```javascript
// 判断这一列是不是"今天"
var isTodayCol = (currentViewWeek === CURRENT_WEEK) && (dayNum === CURRENT_DAY);

// 给今天的列加上高亮 class
bodyHTML += '<div class="day-cell' + (isTodayCol ? ' today-col' : '') + '">';
```

效果：6 月 12 日是周五 → 周五列自动蓝色高亮。6 月 13 日是周六 → 周六列自动高亮。不需要手动指定。
