# 高校教务系统学生端 — 设计说明文档

---

## 一、设计背景与目标

### 1.1 传统教务系统的核心痛点

经过对国内多所高校现有教务系统的调研，我们归纳出以下五个普遍问题：

**信息过载，认知负荷高。** 传统系统将课表、成绩、通知、考试等功能平铺在深层菜单中，学生需要多次点击才能获取"今天有什么课""下周有什么考试"这类高频信息。首页往往是一张空白的公告栏或功能列表，缺乏信息聚合能力。

**操作路径冗长，效率低下。** 选课场景最为典型——搜索课程需要反复切换检索条件，每选一门课要经历"搜索→点击→确认→返回"的四步循环，热门课程在操作间隙已被抢光。

**视觉陈旧，缺乏层次。** 大量系统仍沿用十年前的表格布局和默认配色，不同模块之间边界模糊，重要信息（考试倒计时、选课截止）与普通通知混排，学生需要用"肉眼扫描"代替"视觉引导"。

**移动端体验缺失或敷衍。** 部分系统未做响应式适配，手机查课表需要反复缩放拖拽；即使有移动版，也往往是在桌面版基础上简单压缩，交互逻辑未针对触屏优化。

**缺乏个性化反馈。** 学生的 GPA 趋势、学分缺口、课程冲突等信息散落在多个子系统中，没有一个统一的"学业仪表盘"让学生一眼看清自己的学业状态。

### 1.2 设计目标

基于上述痛点，本设计确立了四个核心目标：

1. **3 秒原则**：学生打开系统 3 秒内，能获取"今天要做什么"的完整答案
2. **操作减半**：选课、查成绩等高频操作的点击步数减少 50% 以上
3. **视觉降噪**：通过色彩、间距、卡片的层级管理，让重要信息自然浮现
4. **全设备一致体验**：桌面、平板、手机三端统一设计语言，核心功能零适配损失

## 二、核心设计理念：以「此时此刻」为中心的信息降噪

### 2.1 设计哲学

传统教务系统是"功能驱动"的：系统有什么功能，就列出什么菜单。我们改为**"场景驱动"**：学生此刻最关心什么，什么就放在最显眼的位置。

这一理念在首页 Dashboard 的设计中体现得最为充分。首页不是一个功能入口堆，而是一个**"个人教务状态全景快照"**。它回答六个问题：

| 问题 | 对应卡片 | 信息层级 |
|:---|:---|:---|
| 现在在上什么课？ | 今日课表快照（最大卡片） | 最高 |
| 最近有什么考试？ | 考试倒计时 | 高 |
| 学分修够了吗？ | 学分达成环形图 | 中 |
| 有什么新通知？ | 最新通知列表 | 中 |
| GPA 怎么样？ | 趋势折线图 | 中 |
| 还有什么待办？ | 快捷操作 + 待办清单 | 辅助 |

### 2.2 Bento Grid 便当盒布局

首页采用非对称的 CSS Grid 便当盒（Bento Grid）布局——大卡片与小卡片混排，不同信息根据重要性被赋予不同的视觉权重。今日课表占据 2×3 格，是视觉锚点；考试倒计时和通知各占 1×1 格，紧凑高效；GPA 趋势横跨全宽，提供数据纵深。

这种布局的核心理念是"信息不平权"——重要的信息占据更大的空间，而不是所有模块均分画布。

### 2.3 课表：最高频场景的极致效率

课表页是整个系统使用频率最高的页面（学生每天打开 3-5 次）。我们在设计上做了三个关键决策：

**三态视图切换**："今天"视图用于早上出门前快速确认；"三天"视图用于规划短周期；"整周"视图是默认态，提供全局视野。三种视图一键切换，信息密度随场景自适应。

**动态日期高亮**：系统根据真实日期自动计算当前教学周和星期几，当天的列自动高亮蓝色底色。6 月 12 日打开就是周五高亮，明天 13 号打开自动变周六——学生不需要手动定位"今天是周几"。

**课程状态感知**：课程格子根据当前时间自动标注状态——🟢进行中（蓝底+红色脉冲点）、🔵待上课（白底蓝边）、⚪已结束（灰色半透明）。学生扫一眼就知道"这堂课已经上过了吗"。

## 三、视觉规范亮点

### 3.1 色彩体系

```
主色：学术蓝 #3B82F6
强调色：灵感金 #F59E0B
中性色：Slate 板岩灰（带蓝紫底调）
```

**为什么选蓝色？** 蓝色在心理学上关联信任、专注、秩序——恰好是教务系统需要传递的情感。不同于金融蓝的"保守"，`#3B82F6` 饱和度略高，带有年轻活力。高校 .edu 域名的视觉基因本身就是蓝色（清华、北大、哈佛均以蓝为基调），学生用户有天然的认知惯性。

**为什么用金色作为强调色？** 蓝色承载 90% 的日常 UI，金色只在"需要学生关注"的地方出现：考试倒计时 ≤3 天、学分缺口预警。稀缺性使用 = 天然的注意力权重。金 = 勋章 / 成就——用金色标注"考试"和"学分"暗示这是学业里程碑，而非用红色制造焦虑。

**为什么用 Slate 灰而非纯灰？** 纯灰色 `#888888` 显得死板。Slate 色系（如 `#64748B`、`#334155`）带有极微妙的蓝紫底调，与蓝色主色同源呼应——这是让界面"不死板"的关键技巧。

### 3.2 字体与排版

| 层级 | 中文 | 英文/数字 | 理由 |
|:---|:---|:---|:---|
| 正文 | PingFang SC / 微软雅黑 | Inter | Inter 的 x-height 极高，小字号下可读性远超 Helvetica；数字 `1` 有明确基座，适合 GPA、学分、日期 |
| 代码 | — | JetBrains Mono | 课程代码如 `CS201` 使用等宽字体 |

**字号层级**：从 32px（首页核心数字）到 11px（辅助标签），7 级字号覆盖全部信息密度需求。大字号行高紧缩（`1.2`），小字号行高宽松（`1.5`），遵循传统印刷排版规则。

### 3.3 卡片与圆角

- **16px 圆角**（默认卡片）：现代 SaaS 产品的"黄金圆角"——圆润但不幼稚
- **10px 圆角**（课程格子）：紧凑网格中需要更明确的边界
- **1px 边框 + 极淡多层阴影**：避免纯阴影卡片在密集网格中边界模糊

Hover 态的最内层阴影用蓝色 `rgba(59,130,246,0.08)` 替代纯黑——用户不会注意到蓝色，但会觉得"这个 Hover 很舒服"。

### 3.4 暗黑模式

使用 CSS Variables 全局切换。暗黑模式下蓝色、金色向浅色方向偏移 100-200（保持视觉亮度感知一致），中性色翻转明暗。所有过渡带 `transition: color 0.3s, background-color 0.3s`，切换时不闪烁。

## 四、交互创新点

### 4.1 选课购物车 + 提交 / 取消 / 退选三态流程

**问题**：传统选课是"搜一门 → 点选 → 确认 → 搜下一门"的串行操作，每多一个步骤就多一分热门课被抢光的风险。

**方案**：引入电商购物车模型——学生在选课开放前即可将心仪课程加入购物车，排好优先级（拖拽排序）。选课开放瞬间点击「一键提交」，系统按优先级批量提交。提交后还可「取消提交」恢复余量，或逐门「退选」。

**技术实现**：选课状态通过 `localStorage` 三重持久化（购物车 + 已提交 ID + 课程余量），切换页面不丢失；提交按钮挂载 500ms 防抖防止重复点击；提交成功时课程余量实时 -1。

### 4.2 通知「稍后再读」+ 置顶机制

**问题**：传统通知列表是静态的——要么全读、要么全不读。学生看到一条重要通知但不能立即处理时，只能记住"回来看"，但往往忘记。

**方案**：每条通知卡片右下角有「稍后再读」按钮。点击后该条立即置顶到列表顶部，左侧变为金色边框，上方显示 📌 标记。当学生点击阅读后，自动取消置顶并回归原位。如果通知已读但又想重新关注，再次点击「稍后再读」会恢复为未读状态。

**设计意图**：这不是简单的"标记"，而是一种**轻量级的任务管理**——把通知列表变成了临时的待办清单。置顶状态也通过 `localStorage` 持久化，切换页面不丢失。

### 4.3 全局搜索浮层

按 `Ctrl+K` 或点击顶栏搜索框，弹出搜索浮层。索引覆盖所有功能页面（12 个）、热门课程、最新通知共 22 条记录，实时过滤，点击结果直接跳转。这在桌面端是效率工具，在答辩展示时是"这个系统功能很多，但都能搜到"的自信证明。

## 五、技术实现概要

| 技术点 | 实现方式 |
|:---|:---|
| 设计系统 | CSS Variables 全局 Token，13 个页面统一引用 |
| 暗黑模式 | `body.dark` 类名切换 + CSS Variables 覆盖 + localStorage |
| 数据可视化 | ECharts 5，共 12 个图表实例（折线/柱状/饼图/雷达/仪表盘） |
| 跨页面状态 | 8 个 localStorage 键值对（主题/未读数/已读状态/姓名/购物车/余量等） |
| 响应式 | 4 档断点（480/768/1024/1280），CSS Grid 自适应 + `_shell.css` 全局补丁 |
| 动效 | Cubic-bezier 弹性曲线、骨架屏脉冲波、页面切换淡入上浮 |
| 防抖 | 搜索输入 250ms、选课提交 500ms |
| 图标 | Lucide 线性图标库，全站 `stroke-width: 1.75` |

全部代码使用原生 HTML/CSS/JavaScript，不依赖任何 UI 框架（Tailwind/Bootstrap），满足课程"纯手搓"的考核要求。

---

## 六、总结

本设计从"学生在什么场景下最需要什么信息"出发，将传统教务系统从"功能列表式的后台工具"重构为"以个人学业状态为中心的智能仪表盘"。在视觉层面建立了一套完整的 Design System（色彩 / 字体 / 圆角 / 阴影 / 动效），在交互层面引入了购物车式选课、稍后再读置顶等创新模式，在技术层面用原生前端技术栈实现了 localStorage 跨页面状态管理、ECharts 多维数据可视化和四档响应式适配。

整套系统共 13 个页面、约 8,500 行代码，覆盖了高校教务系统学生端的全部核心场景。
